# Auto-Correct: Vehicle Inspection Dashboard

Auto-Correct is a web-based dashboard prototype designed for monitoring and analyzing vehicle inspection operations.  
The project focuses on station workload tracking, customer flow analysis, EV safety monitoring, short-term demand forecasting, and operational decision support.

## Live Demo

The project can be viewed through GitHub Pages:

https://huseyincanergin.github.io/Auto-Correct/

## Project Overview

Vehicle inspection stations often deal with fluctuating demand, waiting queues, service capacity limits, and operational bottlenecks.  
This project presents a dashboard-based decision support system that helps visualize these operational dynamics in a more understandable and interactive way.

The system includes multiple dashboard pages such as general overview, station map, EV safety analysis, customer analysis, simulation, and report generation.

## Main Features

- Real-time style vehicle inspection dashboard
- Station-based workload and queue monitoring
- City and station selection with dynamic map visualization
- Short-term station density forecasting
- Customer analysis with complaint, service, vehicle, and payment details
- EV safety analysis with battery health and risk indicators
- Operation simulation for demand and capacity planning
- AI-style recommendation and analysis panels
- Turkish and English language support
- Dark and light theme support
- GitHub Pages compatible single-page HTML structure

## Dashboard Pages

### 1. General Overview

The overview page summarizes the daily operation with key indicators such as:

- Total inspected vehicles
- Average waiting time
- Active inspection capacity
- EV inspection ratio
- Station workload status
- Live demand flow

### 2. Station Map

The map page allows users to monitor stations by city and region.  
Each station includes workload, vehicles under inspection, waiting vehicles, estimated waiting time, personnel count, and daily inspection numbers.

### 3. EV Safety Analysis

This page focuses on electric vehicle inspection records.  
It includes battery health, thermal risk, insulation status, OBD/BMS compatibility, and AI-style risk analysis.

### 4. Customer Analysis

The customer analysis page displays completed inspection records.  
Each record includes customer information, vehicle details, complaint type, performed operations, payment amount, and recommended follow-up actions.

### 5. Simulation

The simulation page estimates future station density based on selected city, station, and time horizon.  
It helps compare expected incoming vehicle traffic, workload, waiting queue, and short-term operational risk.

### 6. Reports

The reports page provides generated operational summaries and downloadable-style report cards for decision support.
